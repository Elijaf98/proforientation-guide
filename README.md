# 🧭 Профориентация

> **Пять вопросов — и ты знаешь, в какой IT-профессии тебе будет по кайфу.**

Мини-тест, который не гадает по звёздам, а считает. Пять коротких вопросов, живые формулировки без HR-канцелярита — и на выходе одно конкретное направление с объяснением «почему именно оно» и первым шагом, который можно сделать уже сегодня. Второй вопрос подстраивается под твой первый ответ, баллы честно суммируются по шести IT-профессиям, а результат сохраняется — чтобы при следующем входе ты сразу видел своё заключение, а не проходил всё заново.

## 🎯 Что проходит пользователь

- **🔓 Вход за 10 секунд.** Авторизация по email и паролю. Регистрация открытая, без подтверждения почты — любой создаёт аккаунт и проходит весь путь сам, без приглашений и ожидания письма.
- **🌿 Опрос с ветвлением.** Пять вопросов, но это не линейная анкета: **второй вопрос зависит от первого**. Выбрал «работаю с данными» — Q2 копнёт глубже именно в данные. Никаких вопросов «не в тему».
- **🎓 Конкретный итог.** В конце — одно подходящее направление, объяснение, почему оно тебе подходит, что подтянуть и какой сделать первый шаг. Плюс наглядные баллы по всем направлениям.
- **💾 Память между визитами.** При повторном входе показывается твоё прошлое заключение — не нужно проходить тест заново.
- **🗄️ Всё сохраняется.** И ответы, и итоговый результат пишутся в базу — историю прохождения видно, и она привязана к аккаунту.

## 🧠 Как работает логика

Никакой магии — прозрачный скоринг по **6 направлениям**:

| | Направление | Про что оно |
|---|---|---|
| 📊 | **Аналитик данных** | порядок в цифрах, поиск закономерностей, дашборды |
| ⚙️ | **No-code / автоматизатор** | связки сервисов, боты, «чтобы процесс шёл сам» |
| 🚀 | **Продакт-менеджер** | «кому и зачем нужно», гипотезы, метрики продукта |
| 🗂️ | **Проджект / операционный менеджер** | статусы, сроки, договорённости, процессы |
| 📈 | **Маркетолог-аналитик** | воронки, эксперименты, рост по метрикам |
| 🔍 | **QA / контроль качества** | внимательность к деталям, «чтобы всё сходилось» |

**Как считаются баллы:**

1. **Вопрос 1** задаёт основной трек — *данные / автоматизация / продукт / люди* — и определяет, какая ветка покажется дальше.
2. **Вопрос 2** ветвится под выбранный трек и углубляется именно в него.
3. **Вопросы 3–5** общие для всех — добавляют объёма картине.
4. Каждый ответ начисляет баллы одному или нескольким направлениям. В конце они **суммируются**, и направление с максимумом становится результатом.
5. **При равенстве** действует фиксированный приоритет:
   > 📊 Аналитик → ⚙️ No-code → 🚀 Продакт → 🗂️ Проджект → 📈 Маркетолог → 🔍 QA

## 🗄️ Схема базы данных

Две таблицы в PostgreSQL (Supabase). Результат и ответы связаны по `result_id`.

### `quiz_results` — итоговое заключение

| Колонка | Тип | Назначение |
|---|---|---|
| `id` | `uuid` (PK) | идентификатор результата |
| `user_id` | `uuid` | владелец результата |
| `top_direction` | `text` | ключ победившего направления (`analyst`, `nocode`, …) |
| `top_title` | `text` | человекочитаемое название направления |
| `scores` | `json` | баллы по всем 6 направлениям |
| `conclusion` | `text` | текст заключения |
| `created_at` | `timestamp` | когда пройден тест |

### `quiz_answers` — отдельные ответы

| Колонка | Тип | Назначение |
|---|---|---|
| `id` | `uuid` (PK) | идентификатор ответа |
| `result_id` | `uuid` (FK → `quiz_results.id`) | к какому результату относится |
| `user_id` | `uuid` | владелец ответа |
| `question_key` | `text` | ключ вопроса (`q1`…`q5`) |
| `question_text` | `text` | текст вопроса на момент прохождения |
| `answer_text` | `text` | выбранный вариант |
| `created_at` | `timestamp` | время ответа |

🔒 **Row Level Security включён** — на уровне базы каждый пользователь видит и пишет **только свои** записи. Чужие результаты недоступны даже при прямом запросе.

> 📬 **Доступ для проверки.** Чтобы посмотреть базу изнутри, доступ выдаётся приглашением в проект Supabase на почту проверяющего.
