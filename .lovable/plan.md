## Что строим

Мини-приложение профориентации на TanStack Start + React + Supabase. Сейчас — каркас с заглушками; реальную логику расчёта направления добавим следующим шагом.

## Авторизация

- Email + пароль через `supabase.auth.signUp` / `signInWithPassword`.
- Без подтверждения почты. Подтверждение отключается в настройках Supabase Auth (нужно будет включить «Allow new users to sign up» и выключить «Confirm email» в дашборде — об этом напомню после реализации).
- Маршрут `/auth` — публичная страница входа/регистрации (табы «Войти» / «Регистрация»).
- Защищённая зона — под `src/routes/_authenticated/` (managed-гейт уже есть в стеке: редирект на `/auth` если нет сессии).
- Кнопка «Выйти» в шапке защищённой зоны.

## Экраны и маршруты

```
/auth                              — вход/регистрация
/_authenticated/                   — стартовый экран (описание + «Начать»)
/_authenticated/quiz               — опрос (вопросы по одному + прогресс)
/_authenticated/result             — экран результата
```

Состояние прохождения теста (текущий вопрос, ответы) — локальное в компоненте quiz (`useState`), затем при завершении передаётся в `/result` через router state / search params. Запись в Supabase (`quiz_results`, `quiz_answers`) — на следующем шаге вместе с реальной логикой.

### 1. `/auth`
Карточка по центру, два таба: «Войти» / «Регистрация». Поля email + пароль, кнопка submit, ошибки через `sonner`. После успеха — `navigate({ to: "/" })`.

### 2. Стартовый `/`
Заголовок «Профориентация», 2–3 строки описания теста, крупная изумрудная кнопка «Начать тест» → `/quiz`.

### 3. Опрос `/quiz`
- Заглушки: 5 вопросов, у каждого 4 варианта ответа (тексты-плейсхолдеры).
- Шапка: «Вопрос X из 5» + прогресс-бар (shadcn `Progress`).
- Карточка с текстом вопроса и 4 крупными кнопками-вариантами (по клику — сразу следующий вопрос, ответ сохраняется в локальный массив).
- На последнем вопросе → переход на `/result` с передачей ответов.
- Кнопка «Назад» к предыдущему вопросу.

### 4. Результат `/result`
- Заглушка направления (например, «Аналитик данных»), короткое объяснение и список из 3 рекомендаций (профессии / шаги развития).
- Кнопки «Пройти ещё раз» → `/quiz` и «На главную» → `/`.

## Дизайн

- Светлая тема, акцент — спокойный изумрудный (`oklch` token `--primary` ≈ emerald 600).
- Обновлю токены в `src/styles.css`: `--primary`, `--ring`, мягкие тени `--shadow-soft`.
- Карточки `rounded-2xl`, `shadow-soft`, обильные паддинги.
- Крупные кнопки-варианты: full-width, `h-auto py-4`, лево-выравненный текст, hover — лёгкая подсветка изумрудным.
- Адаптив: контейнер `max-w-xl mx-auto px-4`, всё работает на мобильном по вертикали.
- Шрифт: системный sans + Inter как body, заголовки — крупный жирный sans (подключу `<link>` к Google Fonts в `__root.tsx`).
- Все тексты на русском.
